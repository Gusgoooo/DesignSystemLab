# Tables

Tables are card-like previews without card background rendering.

## Row Purpose

Each row should expose enough identity, status, key facts, latest signal, and action context for the user to decide whether to open it.

## Column Order

Use progressive exploration:

1. identity
2. status or category
3. most decision-relevant facts
4. latest signal or next action
5. row actions

## Scannability

Do not dump every available field into columns.

Move deeper information into:

- row expansion
- sheet
- detail page
- popover
- tooltip

Use:

- concise cell layouts
- secondary metadata lines
- badges
- compact value stacks
- truncation with accessible full-content access

Avoid paragraph-heavy cells.

## Numeric Tables

Use:

- consistent alignment
- units
- tabular numbers when available
- grouped metrics when several values belong together

## Visual Hierarchy

Table hierarchy should come from:

- typography
- alignment
- spacing
- dividers
- status badges
- row states

Do not turn tables into decorative card backgrounds.

## States

Row hover, selected, focused, disabled, loading, empty, and error states must remain readable and token-correct.
