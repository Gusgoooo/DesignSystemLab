# Visual QA Gate

Use this before final report.

## Better-Than-Before Check

Confirm:

- the primary task is easier to find and complete
- information hierarchy is clearer
- arbitrary surfaces and visual effects were reduced
- spacing, radius, and state behavior are more consistent
- component normalization preserved or improved the workflow
- token pairs are correct on filled surfaces
- responsive states do not overflow

If the answer is no, simplify before finishing.

## Report Format

Report:

- visual structure changed
- page or component patterns unified
- old UI debt removed or reduced
- workflows unchanged
- token pairs checked
- responsive states checked
- remaining pages or components needing migration
